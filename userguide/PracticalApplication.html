Feature Testing with Crawljax and JavaScript

Practical Applications and Future Work

This software has several interesting practical applications. Since the CrawlJax test server itself is independent of the User Story Wiki, it can be used as a generalized engine for testing arbitrary AJAX enabled web applications. The Sample User Story Wiki is one example of how you could visualize test progress over well defined User Stories, all on an external site - meaning testers and developers can work independently if they so choose.

One of the drawbacks of such an approach is the ad-hoc nature of testing. Some integrated web application testing frameworks (like Spec for Ruby on Rails) allow you to write tests very quickly from the comfort of your development environment. Software such as the Sample User Story Wiki would be more useful for Black Box user interface testing and visualizations of project completion based on the tests done directly on User Stories. A related advantage of this approach is that the testing environment becomes language/development environment agnostic (so long as the application you are testing can be accessed entirely from the browser, with some limitations). I tested a PHP application (WordPress), but even a simple static HTML site could have been traversed and tested.

Future Work

  • More error checking. Right now if a script dies, no callback is made to the server; the test case is aborted.
  • Implement Crawler.input(element,"value") (in CrawlJaxServer's crawljax.js)
  • Extend the Crawler.click identification structure (right now identifies only by tag name and id)
  • Add a JavaScript code editor in place of the textarea in the Sample User Wiki
  • Better separation of test cases (allow more than 1 large block test case) from User Stories

Known Issues

  • A limitation of the selenium WebDriver used by Crawljax does not allow for the web browser to respond to requests for Basic or Digest HTTP Authentication
  • Testing applications with invalid Certificate Authorities (i.e. ones that warn of CA invalidity) is currently not supported
  • Clearing Test Cases sometimes doesn't remove orphaned errors from the database
  • Occasional erroneous behaviour on click from CrawlJax (I think the configuration needs to be reset in between invocations)
